Git

Populiariausia kodo versijavimo programa.

Git komandos

  • git init - sukuria lokalią git'o repozitoriją dabartiniame aplankale.
  • git add - prideda nurodytus failus prie commit'ui paruoštų failų (stage'ina).
  • git commit -m "message" - sukuria kodo versiją iš stage'intų failų ir prideda privalomą žinutę apie pakitimus(žinutė turėtų būti ne ilgesnė nei 50 simbolių).
  • git status - nurodo informaciją paie tavo lokalios git'o repozitorijos statusą.
  • git push - iš lokalios repozitorijos išsiunčia commit'us į išorinę repozitoriją.
  • git clone - klonuoja nurodytą repozitoriją iš išorinės repozitorijos.
  • git pull - iš išorinės repozitorijos atsiunčiami commit'ai į lokalią repozitoriją.
  • git log - grąžina commit'ų sąrašą.
  • git checkout branchName | commitID - pereinama į nurodytą šaką arba kodo versiją.
  • branch naujosSakosPavadinimas - sukuria naują šaką (branch).

Branch protection rules

Viena svarbiausių šakų apsaugos taisyklių yra Pull Request nustatymas.

Pull Request eiliškumas

  1. Susikuriama nauja šaka - git branch newBranchName;
  2. Nueinama į naują šaką - git checkout newBranchName;
  3. Atlieki užduotį, kurią apsiėmiai atlikti toje šakoje;
  4. Add or commit atliktus veiksmus(nebūtinai vienas commit'as);
  5. Nueiti į main branch - git checkout main;
  6. Atnaujinti main branch - git pull;
  7. Nueiti į susikurtą šaką (branch) - git checkout newBranchName;
  8. Sujungi savo branch su main branch ir išsprendi galimus konfliktus - git merge main;
  9. Nusiųsti naujos šakos pakitimus į gitHub'ą - git push;
  10. GitHub sukurti Pull Request
  11. Kiti komandos nariai patvirtina Pull Request;
